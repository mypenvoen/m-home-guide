# m-home-guide

## 项目简介

m-home-guide 是一个用于归档和发布多个独立 HTML 页面的仓库。这里存放的页面内容涵盖工具、信息、导航等常见用途，每个页面均为独立的静态 HTML 文件，可直接通过浏览器访问。

本仓库不针对任何特定网站或域名，仅作为页面资源的集中存放与版本管理地。

## 目录说明

```
m-home-guide/
├── pages/          # 存放所有独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/         # 页面引用的公共资源（如 CSS、图片等）
│   ├── css/
│   ├── img/
│   └── ...
└── README.md       # 本文件
```

- `pages/`：每个独立功能对应一个 HTML 文件，文件名通常反映其用途。
- `assets/`：存放页面间共享的样式、脚本或图片资源，避免重复。

## 页面归档说明

- 每个 HTML 页面均为自包含的静态文件，可直接在浏览器中运行。
- 页面内容力求简洁、清晰，不依赖外部服务或动态后端。
- 归档时尽量保持原始结构，如需引用资源，优先使用仓库内的 `assets/` 目录。

## 维护说明

- 所有页面更新、新增或删除均通过 Git 提交记录管理，方便回溯。
- 欢迎提交 Issue 或 Pull Request 以改进页面内容或结构。
- 请勿在页面中插入任何形式的跟踪、广告或恶意代码。
- 本仓库仅用于页面归档与发布，不提供任何形式的服务保障。

## 使用方式

1. 克隆或下载本仓库。
2. 直接打开 `pages/` 目录下对应的 HTML 文件。
3. 或通过 GitHub Pages 等服务进行静态部署。

## 贡献

如果你有好的页面想法或改进建议，欢迎提交 PR。请确保：

- 页面为独立 HTML 文件，无外部依赖或仅引用 `assets/` 中的资源。
- 文件命名清晰，便于识别用途。
- 不包含任何违反法律法规或社区规范的内容。
